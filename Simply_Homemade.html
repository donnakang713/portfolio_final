Life gets busy. Who has time to cook? Try Simply Homemade.

Simply Homemade is the next “Uber of home cooked meals” where users can buy and sell home cooked meals with just a few taps on the screen. It’s easy, it’s fast, and meals are simply homemade.

Problem

As our society gets busier, preparing daily meals become a chore. Thankfully, in the age of apps, one can order healthy and delicious meals for delivery with just a few taps on the screen. I wanted to design a mobile application for the next “Uber of home cooked meals” where users buy and sell home cooked meals. This required me to create buyer & seller user dashboards and user experiences that make it easy for users to explore, buy, and sell home cooked meals.

Solution

Simply Homemade makes finding a great meal super easy and affordable by connecting users with local cooks who want to share their passion for home cooking with the people in their community. Simply Homemade helps busy people eat smarter and live more. It also provides an outlet for talented and passionate chefs to be creative with their meals and get paid to do what they love.

User Survey

When it comes to creating a great product, it’s all about the users. I created a user survey to better understand my potential users and their opinions on meal delivery applications. The goal of the survey was to find out the following:

  • Reasons for using and not using meal delivery applications
  • Which brand is their most and least favorite and why?
  • What features do users like to see in this application?
  • Information on Seller: Years of cooking experience, and who they normally cook for.
  • Information on Buyer: User’s relationship to cooking and how often they cook.
  • Demographics info: Age range, marital status, employment status, and living situation

Analyzing Results

I took the time to analyze my results from user survey, competitive analysis, and user personas. This gave me a clearer vision for my project:

Target Audience

This app will target busy young professionals, students, and young families to easily find meals provided in their local area. Anyone can apply to become a chef with a simple 3-step application process.

Differentiators

  • Affordable homemade meals (As low as $5/serving)
  • Ability to Filter views for easier search
  • Partnership with Uber Eats - Chefs don’t have to deliver
  • Ability to see all ingredients and reviews/ratings of a meal
  • Ability to track your delivery
  • Easy 3-step application for becoming a chef

User Stories & User Flows

After analyzing my results from previous steps, I moved forward with creating user stories and user flows. I selected 5 user stories to highlight the main features of the application and converted them into user flows to show how the user would interact with the application.

View User Flows in detail

User Flow 1: Onboarding Process

As a new buyer, I want to sign up so that I can place meal orders.
As a returning buyer, I want to log in so that I can place meal orders.

User Flow 2: Purchase a meal

As a returning buyer, I want to place an order so that I can make a purchase.

User Flow 3: Apply to become a chef

As a new seller, I want to apply to become a chef so that I post my meals.

User Flow 4: Post a meal

As a returning seller, I want to post a meal so that I can make a sale.

User Flow 5: Edit Chef profile

As a returning seller, I want to edit my profile so that buyers get to know me.

Wireframes

With my five main user stories and flows in place, I began writing a detailed content strategy document outlining the text needed to build a prototype. I then created pre-wireframe sketches for each user flow. Using Sketch, I was able to create a clear layout with all the necessary elements included and located things that do not make sense.

View Wireframe Prototype

I tested my wireframes using Maze. View my test here. I received a lot of comments on the “Purchase a meal” and “Apply to become a chef” user flows. Users wanted to page to view order details before they actually place a meal order. Users also noted that the chef application process was too long and it consisted of repetitive questions. Users wanted it to be simpler and concise.

High-Fidelity Mockups

Below are some different high-fidelity design mockup versions of Simply Homemade, paired with comments from user tests and updated versions.

Buyer's Dashboard

    Problem:
  1. “Recents icon (located bottom center) does not seem to be as useful.”
  2. “I would like to see list and grid view of this page.”
  3. “How can I be able to save meals to my favorites page?”
    Solution:
  1. I removed Recents icon and included Shopping Cart icon, which allows users to purchase all meals at once.
  2. I included Heart icons for each meal picture. User can save meals in Favorites view.
  3. I included option for list view and grid view.

Preview post before posting a meal

    Problem:
  1. “Instead of having the option to click ‘Preview’ can it just show the preview?”
  2. “It’s always good to see preview page before I post.”
  3. “I would like to be able to edit my post.”
    Solution:
  1. I created a page that directly guides the user to the preview page (without having to click preview button)
  2. I included “Edit post” and “Post meal” buttons.

Apply to Become a Chef

    Problem:
  1. “Edit buttons are too distracting with the main call-to-action button under the Background Check section”
  2. “Questions in 'Create a sample dish' process appears to be repetitive. Simply have users upload pictures of their dish.”
    Solution:
  1. I included light arrow icons so that users can easily go back to those sections to edit. Users don’t get distracted to the main background check section.
  2. I removed 'Create a sample dish' questions as it appeared too long and repetitive. Users can simply upload meal pics instead.

Set Meal Availability

    Problem:
  1. This page allows sellers to input their dish availability when posting a meal.
  2. “Date and Time sections appears too vague.”
  3. “How do you input start and end times and start and end dates?"
    Solution:
  1. I included section 1, which allows sellers to select which days of the week they want to provide this dish.
  2. I included section 2, which allows sellers to select daily start and end time.
  3. I included section 3, which allows sellers to select start and end dates.

Meal Info Page

    Problem:
  1. “How can I view order details before I actually click Place an Order?”
  2. “What if I want to change delivery address and payment method?”
  3. “I would also like to be able to ‘add to cart’ so that I don’t have to go through purchase process for each meal”
  4. “Show consistency for the color of the main call-to-action button. In this case, should be turquoise, not orange.”
  5. "The meal is $5 per serving. How do measure what a serving is?"
    Solution:
  1. I included “Add to cart” checkout” button so that user can add meals to cart.
  2. I included “Proceed to checkout” button so that user can review order details and edit delivery address and payment method if need be.
  3. I changed color of the main call-to-action button to turquoise.
  4. Included small icon next to “$5 per serving” so that user can be further informed on how much a serving is.

Branding & Identity

I wanted to leave a positive, friendly, and welcoming vibe for Simply Homemade. I also wanted to let the users know that our chefs are professional and that the meals are healthy and safe to eat. Here is what I did for the branding and identity.

View Style Guide

Logo Design

Simply Homemade’s logo includes a simple wordmark and icon. The steaming pot portrayed as a home represents homecooking and the benefits of it. Meals are healthy and safe to eat - they are “simply homemade”. Lobster is a lovely yet bold and legible script typeface. All together, the logo shows a sense of comfort, warmth, and love.

Typography

Pairing sans serif (Montserrat) with slab-serif (Arvo) work perfectly together especially with contrasting sizes. This set of typefaces best respresent the warm and friendly feel of the brand with a more modern vibe.

Color Palette

Simply Homemade’s color palette is fairly minimal. Orange is a beloved color of food. It is vibrant, friendly, and positive. It’s used as accent colors. Turquoise represents wholeness in our healthy and refreshing meals. It’s used as interactive elements and as background colors. Charcoal and white is used as some typeface colors. Other colors are mostly to be used for accent colors.

Key Learnings

Designing Simply Homemade from start to finish was pure enjoyment and excitement for me. Simply Homemade was inspired by my mother, who is a busy stay-at-home mom and loves home cooking. Here are my key takeaways:

  1. Don't take shortcuts. As a UX designer, it's always tempting to jump ahead to designing mockups without sketching and wireframing. I have all these great ideas and can't wait to get started on designing! However, I learned not to take shortcuts. The more sketching and wireframing I do, I see the value of focusing on the app's functionality and necessary elements.
  2. Put yourself in the user's perspective. Designing Simply Homemade forced me to think in three different points of views: new user, returning buyer, and returning seller. Each of these users have very different objectives when using this app. Taking some time to develop user stories and flows for each user was a vital part.
  3. Listen to your users. There were many instances during the project, where I spent hours figuring out the best designs. Instead of focusing on my own personal preferences, I learned to ask for users' feedback right away. Conducting user tests and preference tests (especially during early wireframe stage) was a vital part of the project.
  4. Keep it simple. The more users I interview, I realized they prefer things to be more simple. Less is more when less does more. Getting rid of repetitive features and confusing elements led to a simple and user-friendly interface.
